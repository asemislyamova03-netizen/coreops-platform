# Implementation Plan: weekend AI life planning content pack

## Goal

Publish the approved Russian weekend/life-planning content pack to the Flexity landing insights, Telegram, and Instagram, while keeping Reels/TikTok draft-only or disabled.

## Classification

- Project: Flexity
- Category: documentation_only
- Risk: medium (static production deploy and two one-time social publications)
- Architecture layer: public marketing/content; no product functionality claim or backend implementation

## Scope

### Files to modify

- `docs/ai/plans/2026-06-27-weekend-ai-life-planning-implementation-plan.md`
- `landing/content/content-packs/2026-06-27-weekend-ai-life-planning/**`

- `landing/www/assets/social/2026-06-27-weekend-ai-life-planning/instagram-feed.png`

### Files not to touch

- `platform-console/**`
- `.github/workflows/**`

- nginx/systemd configuration
- secrets or local credential files
- any other content pack

## Steps

1. Work from a clean worktree based on `origin/main`.
2. Generate and verify the approved 1080x1080 PNG social asset.
3. Add pack metadata and approved Telegram/Instagram text; keep Reels planned only and TikTok disabled.
4. Add the personal reflection article and generate static insights output.
5. Run content tests, compile checks, diff checks, and publisher dry-runs.
6. Commit only allowlisted files and push to `origin/main`.
7. Deploy only `landing/www`, verify public URLs, then publish Telegram and Instagram once through approved workflows when local secrets are unavailable.
8. Do not retry a failed live publish without new explicit approval.

## Tests/checks

- PNG exists, format is PNG, dimensions are 1080x1080.
- `python -m unittest discover -s tests/scripts/content -v`
- `python -m compileall scripts/content`
- `git diff --check`
- `git status --short`
- Telegram dry-run: exactly `Would publish: 1`, `Errors: 0`.
- Instagram dry-run: exactly `Would publish: 1`, `Errors: 0`.
- Public `/insights/`, article, and asset return successful HTTP responses.
- Final git diff contains no forbidden paths.

## Risks

- Generated-image text may require iteration for exact Cyrillic rendering.
- Other approved unpublished packs could make dry-run eligible counts exceed one; stop before live publishing in that case.
- GitHub Actions may advance `main` with publication metadata after each live publish.
- A live API failure must not be retried without explicit approval.

## Rollback

- Before push: discard only this worktree/branch changes.
- After push but before live publish: revert the content commit with a new commit.
- After live publication: do not delete/repost automatically; report the external IDs and request direction.
